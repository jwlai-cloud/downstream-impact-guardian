# examples/

Required by the hackathon submission rules ("Strong submissions include
sample generated artifacts so judges can see the quality of the output").

Once the core loop works, commit 2-3 real generated outputs here directly
(not just referenced from a live demo):
- A real PR comment the agent generated (screenshot or markdown copy)
- The generated compatibility SQL/dbt macro
- The generated test file
- The DataHub Data Contract proposal (screenshot or GraphQL response)

A judge should be able to evaluate output quality from this folder alone,
without running anything.
